üß† Bayesian Optimization Algorithm

üìå Overview

This application implements Bayesian Optimization, a state-of-the-art sequential optimization algorithm designed for expensive black-box functions. It's ideal when each evaluation is costly, time-consuming, or requires manual intervention.

Key Advantage: Bayesian Optimization typically requires 10-100√ó fewer evaluations than grid search or random sampling to find optimal settings.

üî¨ How It Works

The algorithm operates in three main phases:

1. Gaussian Process (GP) Surrogate Model

A probabilistic model that learns the relationship between your DOF settings and outputs. For any untested configuration, the GP predicts both:

  • Expected output (mean prediction)
  • Uncertainty (variance/confidence)

The implementation uses k-nearest neighbor approximation with RBF kernel weighting for computational efficiency.

2. Expected Improvement (EI) Acquisition Function

Intelligently selects the next experiment by balancing:

  • Exploitation: Testing points predicted to be better than current best
  • Exploration: Testing uncertain regions where predictions might be inaccurate

The algorithm computes EI for 100 candidate configurations and recommends the one with highest expected improvement.

3. Sequential Learning Process

Each iteration follows this cycle:

  • Update GP model with all observations
  • Optimize acquisition function to find next best point
  • Recommend DOF settings to user
  • User adjusts system and measures output
  • Algorithm incorporates new data and repeats

üìä Optimization Phases

Exploration Phase (First few iterations): Uses quasi-random sampling to gather diverse observations across the parameter space, helping the GP understand global structure.

Exploitation Phase (Later iterations): Focuses on promising regions identified by the GP, rapidly converging to optimal settings.

üéØ Convergence Criteria

The algorithm declares convergence when:

  • Minimum 10 iterations completed
  • Improvement over last 5 iterations < 1% of target value
  • This indicates the algorithm has found a local optimum

üí° Why This Algorithm?

  • Sample Efficient: Requires far fewer evaluations than alternatives
  • No Gradients Needed: Works with function values alone
  • Handles Noise: Naturally accounts for measurement uncertainty
  • Adaptive Strategy: Automatically adjusts exploration vs exploitation
  • Global Optimization: Not easily trapped in poor local optima
